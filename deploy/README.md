# Production deployment templates

В этом каталоге лежат production-шаблоны и конфиги, которые должны храниться в git.

## Состав каталога

- `apache/000-default.prod.conf` — production-конфиг Apache с HTTP Basic Auth
- `env/.env.prod.example` — шаблон production env-файла

## Назначение

Этот каталог подготавливает репозиторий к будущему production-деплою и автоматизации через Ansible.

Сейчас здесь хранятся только:
- шаблоны
- примеры
- production-конфиги без секретов

## Что НЕ должно храниться в git

В git не должны попадать реальные секреты:

- `.env.prod`
- `.htpasswd`
- реальные production-пароли
- любые локальные файлы с секретами

## Что будет делаться потом, на этапе деплоя

Позже, уже отдельно:
- на основе `env/.env.prod.example` будет создаваться реальный `.env.prod`
- отдельно будет создаваться реальный `.htpasswd`
- `docker-compose.prod.yml` будет использовать эти реальные файлы при запуске production-окружения

## Почему так

Это нормальная схема под будущую автоматизацию:

- в git лежат шаблоны и конфиги
- секреты создаются отдельно
- позже секреты можно будет отдавать через Ansible Vault
